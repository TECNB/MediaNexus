# 接入前端登录态与业务页守卫

## 背景

后端第二批已开始对创建类业务接口要求登录并对普通用户应用每日额度。前端需要让业务页进入前先确认登录态，并提供当前用户展示和登出能力。

## 范围

- 新增前端 auth session provider。
- 启动时根据本地 token 调用 `/api/v1/auth/me` 恢复当前用户。
- 业务布局路由需要登录后访问，未登录跳转 `/login`。
- 登录/注册成功后写入 session 并返回原目标路由或 `/resources`。
- Header 展示当前用户和角色，并提供登出入口。
- Java API 401 清 token 后同步前端 session。

## 非目标

- 不做额度剩余次数展示。
- 不做管理员管理页面。
- 不做 token refresh。
- 不改 Python API 鉴权。

## 验收

- 无 token 访问 `/resources` 会跳转 `/login`。
- 已登录用户刷新业务页会调用 `/auth/me` 并保留页面访问。
- 登录/注册成功后进入业务页。
- 点击登出清理 token 并回到 `/login`。
- 后端 429 错误继续透出到现有页面错误提示。
